Config.json

Este archivo se ubica en styflux/config , yo podría llamar a este archivo el cerebrito del styflux ya que aquí se declaran las variables entre otras configuraciones para administrar a tu necesidad el Styflux.

1. "compile" aquí indicamos donde se compilara(renderizara) nuestros estilos:

"compile":{
	"ext":".styl",
	"cmd":"stylus -c",
	"cmdr":"-o",
	"route":"../css" <!-- esto varía segun nuestra ubicación de carpeta contenedora de css -->
}

2. "layout" contiene la suma de los estilos que estan en styflux/view-helpers:

"layouts":{

"active":true,
"path":"./layouts",

"compile":{
	"layout.styl":{
"helpers":{
"fonts":"fonts.styl", <!-- ejm: @font-face font_face("lucidaRegular") -->
"content":"content.styl", <!-- principales contenedores como: #wrapper .box .main -->
"reset":"reset.styl",
"grids":"grids.styl", <!-- medidas(ancho) recopiladas de nuestro diseño basada en un sistema de grillas -->
"buttons":"buttons.styl",
"forms":"forms.styl", <!-- etiquetas, clases, id que usamos para los formularios en general -->
"table":"table.styl", <!-- clases asignadas a table tr td -->
"plugins":"plugins.styl",
"resources":["resources.styl"], <!-- todas las clases reutilizables -->
"hacks":"[@ult]hacks.styl" <!-- clases para navegador IE -->
}
}
}

3. "views" contiene la suma de los estilos que estan en styflux/views, esto lo usamos en caso de tener 2 tipos de vista para nuestro proyecto, por llamarlo de algún modo "Público" y "Privado":

"views":{
	"path":"./views",
"pathCompile":"./views"
}

4. "vars" en esta variable vamos a declarar ....., adicional a ello indicamos la position X/Y de nuestros icon-sets:

"vars":{
	"styfluxFonts":"[@url]fonts/", <!-- es la ubicación de carpeta contenedora de las fuentes: .eot .svg .woff -->
	"routImg":"[@url]../img/",
	"family1":"'lucidaRegular',Arial,sans-serif", <!-- es recomendable no usar mas de 3 tipos de fuentes en un proyecto -->
	"family2":"'abrazRegular',Arial,sans-serif",
	"family3":"Arial,sans-serif",
	"Styflux":"17px 50%" <!-- en base a un imagen dentro del icon-set declaramos las posiciones x/y en una variable -->
}
